Participants will receive the FedNow broadcast (admi.004) with FNKY code when a new public key is
established. They will then need to download the key via the FedNow interface. Active FedNow public
keys will also be available for download in the Onboarding and Testing section of the FedNow Early
Adopter Hub, under FedNow Service Technical Specifications and Testing Resources (only for CTE).
Participants need to upload the added FedNow public key into their system to ensure that incoming
messages from the FedNow Service can be validated. Participants also need to remove the expired
FedNow public key from their system. As a reminder, we maintain 3 active keys in the Customer Testing
Environment and plan to do so in the Production environment as well. Note that the FedNow Service
typically uses the oldest key for signing messages.
