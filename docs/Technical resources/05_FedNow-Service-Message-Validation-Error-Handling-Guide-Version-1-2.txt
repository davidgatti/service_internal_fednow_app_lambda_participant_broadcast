Introduction
This guide describes the validations run by the FedNow Service and the errors that can occur. Each error condition is fully described and includes
resolutions to help you fix errors. Notes are provided to assist you in understanding the reason the error occurred and where you can find more
information.
Important: This guide does not contain an exhaustive list of all FedNow Service error conditions that can occur. Error conditions that are
anticipated to cause Participants the most difficulty are fully documented below, along with suggested workarounds.
Use this guide with the documents listed below to obtain a complete understanding of the FedNow Service.

Audience
This guide is intended for use by developers at financial institutions (FIs) and Service Providers who use the FedNow Service. References to

To obtain the most benefit from this guide, we recommend adhering to the following guidelines:

2. Next, review the validation overview and checkpoints described in the FedNow Service Message Validation section. Each validation

quick-reference table format that includes: the error code number, the standard error description, error details, resolution and notes. We
recommend reviewing the information for each error condition so you are prepared if issues arise.

FedNow Service Message Validation
When the FedNow Service receives a message off a queue, a series of data validations occurs to ensure the message is secure and viable for
processing. If validation finds a message that is not viable, further validations halt, and in most situations, an admi.002 message is sent to the
sender’s reply queue with the relevant error code and description.
Not all messages sent to the FedNow Service result in a reply. Some messages are dropped. For example, if a message includes an invalid venue
ID, the message is immediately dropped. If a message is put on a queue that does not handle that message type, it is immediately dropped.
If the message is found to be secure and viable, processing proceeds until the message is fully validated.

External Validation Errors
These validation errors are calculated against values that are outside the XML payload, such as queue name and protocol headers.

Message Size Error – T501
Error Description

ISO message exceeds the maximum allowable size of 25,000 characters.

Error Details

The ISO 20022 schemas impose restrictions to message data that will in most cases prevent a message from exceeding the
maximum allowable size.

Resolution

Reduce the message size by removing unnecessary XML elements.

Notes

Minimal viable sample messages are provided in the MyStandards platform that define the XML elements required for
specific use cases.

Message Signing Errors – T529
The T529 error code indicates an error with either the signing key or the message signature. We recommend verifying the signing key first and, if
the signing key is valid, troubleshoot the signature.
Error Description

Invalid message signature on ISO 20022 message. For the signature to be valid, it must be signed with the key owner’s private
key that corresponds to an active public key that was previously created and validated by the FedNow Service.

Error Details

Indicates one of the following errors:

Resolution

•

The queue message header for the signing key ID (fnsigs.028id) is not associated with a signing key in the sender’s
Participant profile or that the key exists but has expired.

•

The queue message header for the message signature (fnsig) must be generated using as inputs: the signing key
associated with the signing key ID queue message header (fnsigid), the signature algorithm specified by the signature
algorithm queue message header (fnsigspec), or the message content. The T529 error code can indicate that this value is
invalid because it cannot be replicated by the FedNow Service using the corresponding signing key stored in the Participant
profile.

•

For a signing key error:
- Make sure you are using an existing and unexpired signing key.
- If an unexpired signing key is not available, add a new signing key to the Participant profile and use that one.
For a message signature error:
- Correct the signature generation code to produce the correct signature.
- Follow the steps for using a valid signing key and try again with a valid signing key.

For Schema Validation Errors – T505
The T505 error code encapsulates all errors that occur during message intake into the FedNow Service. Validations ensure that the message XML
is well-formed, contains only valid XML Unicode characters, and all elements and attributes conform to the FedNow ISO 20222 schema.
Errors that occur at this stage of validation indicate the message is malformed and cannot be parsed. There may be XML elements that are not
properly closed, have misplaced or extra quotes, contain characters that are invalid in an XML document, and other issues that can prevent an
XML document from parsing. A visual inspection often exposes the issue. It also helps to use an XML text editor that can show these types of
issues and offer help in correcting them.
Error Description

ISO message does not meet the FedNow Service ISO 20022 message specifications for one or multiple fields.

Error Details

Common ISO 20022 schema validation errors include:

Resolution

Notes

Scenario 1:

An XML element value does not conform to a w3c XSD datatype.

Scenario 2:

An XML element value does not conform to a regular expression defined in the ISO
schema.

Scenario 3:

An XML element value must be a value from a list of defined values. In such cases, an
enumeration of the eligible values is included in the ISO schema validator error message.

Scenario 1:

Review the w3c XSD datatype and choose a value that conforms.

Scenario 2:

Review the regular expression and choose a value that conforms.

Scenario 3:

Review the eligible values and choose a value that conforms.

Scenario 1:

•
•

This error often surfaces situations where business logic needs to be corrected for things such as
conversion of a date and time to a valid format.
w3c XSD datatype help can be found at XML Schema Date/Time Datatypes (w3schools.com).

Scenario 2:

•

This error often surfaces situations where business logic needs to be corrected for things such as
length of an identifier, a numeric limit, etc.

Scenario 3:

•

This error often surfaces situations where business logic needs to be corrected for the correct
code. You may need to consult with your business analyst for the correct code.
The ISO 20022 public web site publishes enumerations of valid codes with corresponding
descriptions.
The XML schema (XSD) can be consulted for valid codes to ensure the correct code is chosen in

Message sender authorizations determine whether the message sender is authorized to perform the intended transaction.
Error Description

Invalid sending party or sending party information. A valid sending party is a Participant or a Service Provider, authorized to send
messages on behalf of the Participant.

Error Details

•

•

Indicates the Participant who put the message on the queue is not authorized to use the queue or act on behalf of the
transaction owner.
Can also indicate that a key, other than the one derived from a valid signature, has been used.

The system validates that either a master key or the right key for the Participant (target owner) is being used.
Resolution

Notes

Do the following:
•
•

Correct the target queue to a queue authorized for use by a valid sending party.
Review the message values for the connecting party and the transaction owner.

•
•

Error Description

Invalid clearing system member ID field. The ID in the “to” component should be the FedNow Service member ID (e.g.,
"021150706").

Error Details

Indicates that the …/AppHdr/To/FIId/FinInstnId/ClrSysMmbId/MmbId element value does not equal the FedNow Service
clearing system member identification (021150706).

Resolution

Set the …/AppHdr/To/FIId/FinInstnId/ClrSysMmbId/MmbId element value to “021150706.”

Notes

N/A

Message Definition Identifier – T506
Error Description

Invalid message definition identifier. This identifier should be associated with the underlying message and must be a version
that FedNow Service supports.

Error Details

Indicates that the …/AppHdr/MsgDefIdr element value is incorrect for the ISO document type.

Resolution

Set the …/AppHdr/MsgDefIdr element value to the correct ISO document type.

Notes

The FedNow ISO message technical wrapper combines FedNow-specific XML elements with the standard ISO 20022
document elements. The …/AppHdr/MsgDefIdr element value maps the FedNow ISO message envelope to the
corresponding ISO 20022 payload.

Error Details

Resolution

Remove the …/AppHdr/BizPrcgDt element from the message XML.
